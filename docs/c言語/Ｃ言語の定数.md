## 定数の定義について
### なぜ定義するのか
- コードの可読性を上げる
- エラー防止
- 保守性の向上

### 方法1-#define
- プリプロセッサ指令を使用して定数を定義
- コンパイル時に単純な文字列置換を行う
#### 利点
- 簡単に定義できる
- 実行時のオーバーヘッドが少ない
#### 欠点
- 型チェックがない
- デバッグ時に追跡が難しい

例
~~~c
#define PI3.14159
~~~

### 方法2-const 
- 型付き変数を定数として扱うためのキーワード
#### 利点
- 型が存在し、安全性が高い
- デバッグ時の追跡が容易
- スコープの指定が可能
#### 欠点
- 初期が必須で、手間がかかる場合がある

例
~~~c
const int MAX = 500;
~~~

### 方法3-enum
- 関連する整数定数をまとめて定義できる
#### 利点
- 整数定数を一元管理できる
- 可読性、保守性の向上
#### 欠点
- 整数以外の型は扱えない

例
~~~c
enum {
    SUCCESS, // 0
    FAILL, // 1
};
~~~
